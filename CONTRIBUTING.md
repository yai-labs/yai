# Contributing

## Scope

This repo must remain compliant with YAI Law.
Do not introduce changes that alter authority, protocol, or state semantics without law updates.

## Workflow

- Open a PR per change set
- Use descriptive branch names
- Keep changes minimal and scoped

## Tests / Checks

Run the minimal build/test commands for this repo before submitting.

## Compliance

If a change touches ABI, protocol, or state-machine behavior, include a Law Change Record in the PR.
