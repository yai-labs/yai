# D18.2 Cross-Section Authority Contract

## Precedence
1. architecture canonical (model/scope/boundary authority)
2. reference canonical (surface/contract lookup authority)
3. runbooks procedural (execution authority, not design authority)
4. guides support (adoption authority, not canonical authority)
5. program reports (evidence authority, not system authority)
6. archive historical (never authoritative for current state)

## Conflict Resolution Rules
- If architecture and runbook diverge on system model, architecture wins; runbook must be corrected.
- If reference and guide diverge on contract lookup semantics, reference wins; guide must be corrected.
- Program report claims cannot override architecture/reference authority; reports may only summarize evidence.
- Historical docs must never be used as authority for live behavior.
- If two live docs claim the same `primary_for`, the conflict is invalid until one is declassed.

## Divergence Handling Procedure
1. Identify topic and current canonical owner from `d18.2-source-of-truth-contract-map.md`.
2. Move conflicting guidance into owner doc or downgrade conflicting doc role.
3. Update front matter (`role`, `status`, `depends_on`, `superseded_by` where relevant).
4. Update README spines and cross-links.
5. Re-run authority validator before merge.

## Supersession Rules
- A superseded doc must declare `superseded_by`.
- A replacement doc may declare `supersedes`.
- Cross-links must be updated to point to active canonical owners.

## Historical Non-Authority Rule
- Docs under `docs/archive/**` are evidence only and cannot appear in live `depends_on` chains.
