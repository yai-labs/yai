# C17.3 Docs Overlap Audit

## Context

After C17.1 topology refoundation and C17.2 naming cutover, live docs still had heavy thematic overlap and micro-fragmentation.

## Overlap Clusters Identified

1. Workspace architecture cluster
- boundary/containment/scope/isolation/security/state/lifecycle/binding docs overlapped strongly.

2. Distributed runtime + governance cluster
- peering/enrollment/authority/policy distribution/source-plane integration docs overlapped.

3. Developer walkthrough cluster
- multiple walkthrough files covered adjacent runtime-governance operational flows.

4. Runbook scenario cluster
- per-scenario demo/qualification docs duplicated shared procedure skeletons.

5. Program report shard cluster
- many report/matrix variants covered the same convergence themes with minor deltas.

## Corrective Strategy

- Define one canonical source doc per major theme.
- Merge overlapping satellites into canonical spine docs.
- Apply delete-first for absorbed live docs.
- Keep archive only where traceability evidence requires historical retention.
