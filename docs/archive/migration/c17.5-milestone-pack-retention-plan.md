# C17.5 Milestone-Pack Retention Plan

## Policy
- Keep latest only for versioned clusters.
- Remove intermediate versions unless explicit external traceability requires retention.

## Families Reviewed
- contract-baseline-lock
- command-coverage
- engine-attach
- root-hardening
- specs-refactor-foundation
- workspaces-lifecycle
- data-plane

## Decision Matrix
- still active: latest version per cluster + family README
- canonical historical: none in live tree; moved/retained only if evidence-critical
- superseded: intermediate versions
- redundant: duplicate per-wave snapshots not needed for canonical present
- archive-only: only where legal/compliance trace requires

## Retention Rule
Each family README must explicitly state `retention rule: latest-only`.
