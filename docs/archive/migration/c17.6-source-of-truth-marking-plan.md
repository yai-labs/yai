# C17.6 Source-of-Truth Marking Plan

## Role Mapping
- `canonical`: primary source documents for architecture domains.
- `support`: governance/program/guidance support docs.
- `reference`: lookup surfaces for contracts and schemas.
- `procedural`: executable runbook procedures.
- `historical`: archive-only artifacts.

## Status Mapping
- `active`: live and maintained.
- `superseded`: retained for historical reasons only (archive).

## Application Scope
C17.6 applies explicit markers to architecture spines, guide spines, runbook cores, reference entries, and ADR/RFC core documents.

## Editorial Rule
All canonical live docs should expose:
1. Purpose
2. Scope
3. Relationships
4. Canonical Role
5. Main Body
6. Related Docs
