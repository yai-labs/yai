# C17.1 Audience-Function Matrix

## Function Axis

- Architecture: `docs/architecture/**`
- Guides: `docs/guides/**`
- Runbooks: `docs/runbooks/**`
- Reference: `docs/reference/**`
- Program: `docs/program/**`
- Product: `docs/product/**`
- Generated: `docs/generated/**`
- Archive/Historical: `docs/archive/**`

## Audience Axis

- Architecture reviewers / system designers:
  - `docs/architecture/**`
  - selected `docs/reference/**`
- Developers:
  - `docs/guides/developer/**`
  - selected `docs/runbooks/operations/**`
- Operators:
  - `docs/guides/operator/**`
  - `docs/runbooks/**`
- Users:
  - `docs/guides/user/**`
  - `docs/product/scenarios/**`
- Program owners:
  - `docs/program/**`
- Auditors / migration reviewers:
  - `docs/archive/migration/**`
  - `docs/archive/historical-architecture/**`

## Wrong-home patterns removed

- Program baseline execution documents under runbooks.
- Legacy interface notes under active reference.
- Legacy dev-guide drawer under developer guides.

## Classification Rule

A document must be placed by primary use:

1. Canonical model -> architecture/reference.
2. How-to by audience -> guides.
3. Step-by-step execution -> runbooks.
4. Work-governance artifact -> program.
5. Historical/superseded/migration-only -> archive.
