# C17.1 Docs Topology Audit

## Scope

This audit classifies the current `docs/` tree by function and audience, highlights topology collisions, and defines relocation priorities for the canonical hierarchy.

## Macro-area Assessment

- `docs/architecture`: canonical, live, but still mixed with some migration-era references.
- `docs/guides`: canonical, live, but developer content needed decomposition by function.
- `docs/runbooks`: canonical for executable procedures; now reduced to operations/qualification/demos/remediation only.
- `docs/reference`: canonical lookup area; legacy notes removed from live reference.
- `docs/program`: canonical governance-of-work area.
- `docs/product`: canonical product/scenario area.
- `docs/generated`: canonical generated output area.
- `docs/archive`: canonical non-live storage (legacy, migration, historical architecture).

## Topology Collisions Found

1. `runbooks/operations/program` mixed operational runbooks with program baseline packs.
2. `guides/developer/dev-guide` acted as a legacy drawer instead of a structured guide subtree.
3. `reference/interfaces-legacy-notes.md` mixed legacy migration narrative inside active reference.
4. Historical/migration links remained embedded in live architecture and program docs.

## Actions Executed in C17.1

- Collapsed `docs/runbooks/operations/program` and relocated its documents into `docs/program/milestone-packs/runtime-baselines/`.
- Collapsed `docs/guides/developer/dev-guide` into canonical subareas:
  - `build-test/`
  - `workflow/`
  - `tooling/`
  - `governance/`
  - `debugging/`
  - `checklists/`
- Moved `docs/reference/interfaces-legacy-notes.md` to `docs/archive/legacy/reference/`.
- Rewrote path references from removed locations to canonical homes.

## Resulting Topology Quality

- Function-first grouping improved.
- Audience readability improved.
- Live vs historical separation is significantly clearer.
- Remaining debt is primarily semantic deduplication, not topology.
